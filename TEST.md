# 测试迷宫寻路系统

这是一个简单的测试文档，展示如何使用迷宫寻路系统。

## 快速测试

```bash
# 编译程序
make

# 运行程序
./maze_solver
```

## 测试用例

### 测试1：创建简单迷宫
1. 启动程序
2. 选择 "1. 创建迷宫"
3. 选择 "1. 随机生成迷宫"
4. 输入迷宫尺寸：8 12
5. 输入墙壁概率：0.3
6. 选择 "1. 使用默认位置"

### 测试2：使用BFS寻找最短路径
1. 选择 "2. 求解迷宫"
2. 选择 "2. 广度优先搜索 (BFS)"
3. 选择显示路径可视化：y

### 测试3：比较所有算法
1. 选择 "3. 算法性能比较"
2. 观察不同算法的性能差异

### 测试4：导出HTML可视化
1. 选择 "5. 导出结果"
2. 选择 "2. 生成HTML可视化文件"
3. 输入文件名：test_maze
4. 在浏览器中打开 test_maze.html

## 预期结果

程序应该能够：
- 成功生成迷宫
- 找到从起点到终点的路径
- 显示算法性能比较
- 生成HTML可视化文件

## 已知问题

- 某些终端可能不支持颜色显示
- 动画功能需要进一步完善
- 大型迷宫可能需要更长的计算时间

## 性能基准

在标准硬件上（Intel i5，8GB RAM）：
- 10x10迷宫：< 1ms
- 50x50迷宫：< 100ms
- 100x100迷宫：< 1s
